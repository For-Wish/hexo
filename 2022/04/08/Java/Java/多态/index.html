同一个对象，在不同时刻表达出来的不同形态。

多态同一个接口，使用不同的实例而执行不同操作 。

  • 继承
  • 重写

1.5 多态的好处和弊端

  • 多态的好处：提高了程序的扩展性
  • 具体体现：定义方法的税后，使用父类型作为参数，将来在使用的时候，使用具体的子类型参与操作
  • 多态的弊端：不能使用子类的特有功能

1.6 多态的优点

  1. 消除类型之间的耦合关系

  2. 可替换性

  3. 可扩充性

  4. 接口性

  5. 灵活性

  6. 简化性
